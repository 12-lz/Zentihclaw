🔍 Vercel 部署图片问题诊断清单

✅ 第一步：确认问题

🔍 第二步：检查 Vercel Dashboard

📋 如果图片文件不在 Vercel Dashboard 的 Files 中

问题原因：图片文件没有被上传到 Vercel

解决方法：

  1. 确认本地文件夹包含所有文件：
    D:\zenithclaw-vercel\ ├── index.html ├── logo.png ├── qrcode_inverted.jpg └── vercel.json
  2. 重新上传整个文件夹到 Vercel
  3. 或在 Git 仓库中确保图片文件已提交

🌐 如果图片文件在 Vercel Dashboard 中但仍然不显示

可能原因：文件路径问题

解决方法：

A. 使用绝对路径（推荐）

修改 index.html： 将 src="logo.png" 改为 src="/logo.png" 将 src="qrcode_inverted.jpg" 改为 src="/qrcode_inverted.jpg"

B. 检查浏览器控制台

  1. 打开你的网站
  2. 按 F12 打开开发者工具
  3. 切换到 "Console" 标签
  4. 查找红色错误信息
  5. 切换到 "Network" 标签
  6. 刷新页面，查找图片的状态码（200/404）

C. 清除 Vercel 缓存

  1. 访问 Vercel Dashboard
  2. 进入你的项目
  3. 点击 "Settings" → "Functions"
  4. 滚动到底部，点击 "Clear Cache"
  5. 重新部署

🚀 最快速的临时解决方案

使用免费图床托管图片：

  1. 访问 https://imgbb.com
  2. 上传 logo.png 和 qrcode_inverted.jpg
  3. 获取直链（Direct Link）
  4. 修改 index.html，替换图片路径：
<img src="https://i.ibb.co/xxxxx/logo.png" alt="Zenithclaw Logo"> <img src="https://i.ibb.co/xxxxx/qrcode_inverted.jpg" alt="微信二维码">

📊 常见状态码含义

  • 200 - ✅ 成功，图片正常加载
  • 404 - ❌ 文件未找到，路径错误或文件不存在
  • 403 - ❌ 权限问题，无法访问
  • 500 - ❌ 服务器错误
